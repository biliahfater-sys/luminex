# LUMINEX Portfolio

Демонстрационный сайт вымышленной дизайн-студии **LUMINEX** — статическая работа для
портфолио. Реального бэкенда, базы данных и сбора персональных данных нет: форма обратной
связи работает в демо-режиме и никуда не отправляет данные.

Стек: React 19, Vite, TypeScript, Tailwind CSS, GSAP, Framer Motion, Spline, Three.js.

## Запуск на Windows

Дважды кликните `start.cmd` — он проверит Node.js, установит зависимости, соберёт сайт
и откроет статический предпросмотр:

```text
http://localhost:4173/
```

## Скрипты

```bash
npm run dev       # Vite dev-сервер (http://localhost:5173)
npm run build     # Проверка типов + production-сборка в dist/
npm run preview   # Локальный предпросмотр собранного сайта
npm run lint      # ESLint
```

## Сборка и деплой

`npm run build` пишет полностью статический сайт в `dist/`. Эту папку можно положить на
любой статический хостинг (GitHub Pages, Netlify, Vercel, Cloudflare Pages) или встроить
как под-страницу в другой сайт-портфолио. Сервер не требуется.

Маршрутизация — на `HashRouter`, поэтому работает с любого пути без серверных rewrite-правил.

## Правовое (152-ФЗ)

В сайт включены демо-страницы «Политика обработки персональных данных» (`/#/privacy`) и
«Согласие на обработку ПДн» (`/#/consent`), а также всплывающее cookie-уведомление — как
демонстрация требований Федерального закона № 152-ФЗ. На страницах явно указано, что это
демонстрация и реальные данные не собираются. Для «боевого» сайта нужно подставить
реквизиты оператора в `src/lib/legal.ts` и подать уведомление в Роскомнадзор.

Реализовано в демо-режиме:

- отдельная страница политики обработки персональных данных;
- отдельная страница согласия на обработку персональных данных;
- обязательный чекбокс согласия возле формы обратной связи;
- cookie/localStorage-уведомление с явным принятием или отказом;
- честное раскрытие того, что форма статическая и не отправляет данные;
- отсутствие аналитики, рекламных пикселей, внешних форм и серверного хранения.

Перед реальным запуском с обработкой персональных данных нужно:

- заменить демо-реквизиты оператора в `src/lib/legal.ts` на фактические данные;
- проверить хостинг, почту, CRM, аналитику, антиспам и резервные копии на локализацию данных РФ;
- добавить серверную валидацию, rate limit, антиспам и журнал согласий;
- заключить договоры/поручения с подрядчиками, которые получают доступ к данным;
- подать или обновить уведомление в Роскомнадзор, если нет применимого исключения;
- провести юридическую проверку итоговых текстов под конкретного оператора.
